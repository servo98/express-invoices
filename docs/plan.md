# Front setup:

El front estará hecho con las siguientes tecnologías:
- https://ui.shadcn.com/
- nextjs fullstack
- Quiero en todo lo posible una planeación donde la lógica del "negocio" sea agnóstica al código del front que se pueda cambiar o ajusta el diseño ux/ui sin afectar a el negocio y se puedan hacer transiciones más limpias y sin tanto error
- Diseño mobile first


Descripción del proyecto:
El sistema te permita inicar sesión con google

Quiero que el sistema mande mensajes a un canal de discord que se pueda configurar para recordar el hacer la factura de igual manera que al notificar el usuario al dar clic lo lleve al portal donde ya estará la facutura pre hecha

Esta aplicación servirá para poder agilizar el proceso de creación de facturas ante el sat (servicio de administración tributaria mexicana)  

ejemplo de factura en la carpta docs

Después de generar la factura del mes que cree un pdf con una factura parecida al archivo invoice.pdf en la carpeta docs

Que guarde las facturas por mes y por usuario, que cada usuario pueda ver sus propias facturas

Despues de ver la factura y el invoice listos que se pueda descargar como zip la factura en pdf el xml  y el pdf del invoice para enviar posteriormente por correo

Nota importante:
Algo no negociable es que al mandar el recordatorio por ds quiero que lo mande con un ascii de memes del momento y que busque el ascii art automanticamente en internet para dicho mensaje

El diseño técnico del front debe cumplir con un alto estandar como mobile first, modo oscuro, modo pink
